# DrillGuard OS — синхронизация заявки INDUSTRIX 2026

Документ фиксирует соответствие **сопроводительного письма** и репозитория **v0.2.2**.

## Одно предложение

Локальная рекомендательная система раннего выявления **кандидатов** на осложнения при бурении по гидравлическим и механическим сигналам; результат — объяснимая карточка для инженера **без управляющих воздействий**.

## Направление

Бурение и внутрискважинные работы → цифровизация строительства/ремонта скважин → снижение времени первичной инженерной проверки.

## Причинная цепочка

Схема и качество → время → режим → **причинная** базовая линия → отклонения и согласованность → persistence / гистерезис → класс кандидата → карточка → архив → read-only shadow mode.

## Пользователи (письмо)

Инженер по бурению · буровой мастер · сопровождение строительства · диспетчер · специалист по растворам · руководитель программы бурения.

## Входы

- **Обязательные:** timestamp, depth, SPP, flow-in, hookload, torque, ROP, pump rpm, operation, data_quality (`REQUIRED_COLUMNS`).
- **Опциональные (письмо):** flow-out, pit volume, density/rheology (`mud_density_sg`, `plastic_viscosity_cp`, `yield_point_pa`), temperature, cuttings, equipment_status, daily_report_ref, active_alarms, engineer_comment (`OPTIONAL_COLUMNS`).

## Классы результатов (письмо ↔ код)

| Письмо | `EventClass` |
|--------|--------------|
| кандидат на ухудшение очистки / ограничение циркуляции | `possible_packoff` |
| кандидат на поглощение | `possible_lost_circulation` |
| сигнал, похожий на проявление | `possible_influx_candidate` |
| аномалия момента и нагрузки | `torque_drag_anomaly` |
| проблема качества измерений | `sensor_quality_issue` |
| смена операции | `operation_change` |
| короткий выброс | `short_transient` |
| недостаток истории | `insufficient_history` |
| конфликт режима и сигналов | `signal_conflict` |
| отклонение в зоне шума | `normal_noise` / `none` |

## Жёсткие границы

Не управляет буровой · не пишет в АСУ ТП/SCADA · не ПАЗ · не заменяет инженера · не подтверждает аварию/проявление/поглощение/прихват.  
`heuristic_score` — не вероятность. Доказательная база — `synthetic_only` + `requires_field_validation`.

## УТГ / TRL

Самооценка **УТГ 4 / TRL 4** относится к **программному прототипу и воспроизводимому расчётному контуру** (`schema.READINESS_LEVEL`).  
Не означает полевую точность или готовность к промышленной эксплуатации.

## Пилот / shadow (письмо)

Архивная проверка → разметка → holdout → локальный read-only → журналирование карточек → решение продолжить/доработать/остановить.  
См. `docs/PILOT_PLAN.md`, `docs/VALIDATION_PROTOCOL.md`.
