# DrillGuard OS — синхронизация заявки INDUSTRIX

## Одно предложение

Локальная рекомендательная система раннего выявления **кандидатов** на осложнения при бурении по гидравлическим и механическим сигналам; результат — объяснимая карточка для инженера **без управляющих воздействий**.

## Причинная цепочка

Буровые сигналы → качество → синхронизация/время → режим операции → **причинная** базовая линия (только прошлое) → признаки → устойчивость (persistence) → класс кандидата → карточка → ретроспективная проверка → read-only shadow mode.

## Процесс → потеря → пользователь → действие → KPI

| Элемент | Содержание |
|---------|------------|
| Процесс | Бурение / циркуляция (строительство скважины) |
| Потеря (гипотеза) | Поздняя реакция на кандидата осложнения → NPT / риск |
| Пользователь | Инженер бурения / супервайзер |
| Входы | Временные ряды SPP, flow-in, hookload, torque, ROP, operation… |
| Алгоритм | Причинные правила + режимный baseline (без ML в v0.2) |
| Действие | Только проверка по регламенту; **нет** команд насосам/АСУ |
| KPI пилота | FA/час, задержка, полнота/точность на **размеченном** архиве, полезность карточки |

## Что подтверждено (synthetic)

- Воспроизводимый pipeline, причинный baseline, persistence, benchmark с delay/FA/F1, тесты.

## Что не подтверждено

- Полевая точность, экономический эффект, предотвращение аварий, работа на конкретном заказчике.

## Что просим

Обезличенный архив одной серии скважин, словарь тегов/единиц, таксономию операций, эксперта-владельца процесса, согласование ИБ и shadow mode.
